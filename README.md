# IntListPC PF2e

Статическая версия листа персонажа PF2e.

## Файлы

- `index.html` - разметка страницы и подключения.
- `css/styles.css` - все стили, вынесенные из исходного HTML.
- `js/app.js` - вся логика приложения, вынесенная из исходного HTML.
- `assets/icons/favicon.png` - иконка сайта для вкладки браузера.

Из исходного файла удален скрытый хвостовой блок с `Lorem ipsum`, который не участвовал в работе приложения.

## Локальный запуск на ПК

Самый надежный способ для этой папки:

1. Открой папку проекта в терминале.
2. Запусти локальный сервер:

```powershell
powershell -ExecutionPolicy Bypass -File .\serve.ps1
```

3. Открой в браузере:

```text
http://localhost:8000/
```

Если хочешь использовать Python-сервер вместо `serve.ps1`, подойдет так:

```powershell
python -m http.server 8000
```

Если команда `python` не найдена, попробуй:

```powershell
py -m http.server 8000
```

Можно открыть и сам `index.html` двойным кликом, но для теста лучше использовать локальный сервер. Так сайт работает ближе к тому, как он будет работать на GitHub Pages.

## Публикация на GitHub Pages

1. Создай репозиторий на GitHub.
2. Загрузи в него содержимое этой папки: `index.html`, папку `css`, папку `js`, `README.md`.
3. В настройках репозитория открой `Settings -> Pages`.
4. В `Build and deployment` выбери `Deploy from a branch`.
5. Выбери ветку `main` и папку `/root`.
6. Сохрани. Через минуту GitHub даст ссылку вида:

```text
https://USER.github.io/REPOSITORY/
```

Если используешь вход через Google/Supabase, добавь эту ссылку в разрешенные Redirect URLs в настройках Supabase Auth. Для локального теста туда же обычно добавляют:

```text
http://localhost:8000/
```

## Как заменить иконку сайта

Текущая PNG-иконка лежит здесь:

```text
assets/icons/favicon.png
```

Чтобы поставить свою иконку, просто замени этот файл своим PNG. Имя и путь лучше оставить теми же, тогда `index.html` менять не нужно.

Для PNG лучше использовать размер `512x512` или хотя бы `192x192`.
